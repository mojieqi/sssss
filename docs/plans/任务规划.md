  template_type   CHAR(1)      NOT NULL DEFAULT '0'     COMMENT '类型(0系统提示词 1用户提示词)',

### 6.4 记忆机制设计

**短期记忆：**
- 保留最近 N 轮对话（默认10轮）
- 超过限制时，对早期对话自动摘要压缩
- Token 计数和窗口管理

**长期记忆：**
- 提取用户偏好、历史话题、关键信息
- 通过 `ai_user_memory` 表持久化存储
- 每次对话时注入相关长期记忆

```sql
-- 用户长期记忆表
CREATE TABLE ai_user_memory (
  memory_id       BIGINT(20)   NOT NULL AUTO_INCREMENT  COMMENT '记忆ID',
  user_id         BIGINT(20)   NOT NULL                 COMMENT '用户ID',
  memory_key      VARCHAR(100) NOT NULL                 COMMENT '记忆键(话题/偏好/关键信息)',
  memory_value    TEXT         NOT NULL                 COMMENT '记忆内容',
  importance      INT(4)       DEFAULT 1                COMMENT '重要性(1-5)',
  last_access     DATETIME                              COMMENT '最后访问时间',
  access_count    INT(11)      DEFAULT 0                COMMENT '访问次数',
  create_time     DATETIME                              COMMENT '创建时间',
  PRIMARY KEY (memory_id),
  KEY idx_user_id (user_id)
) ENGINE=InnoDB COMMENT='AI用户长期记忆表';
```

### 6.10 技术实现要点

**流式响应：** 使用 Spring `SseEmitter` + `FetchEventSource` 实现 SSE 逐字推送

**Function Calling 循环 (AgentEngine)：**
```
1. 用户消息 → 构建消息列表 (系统提示词 + 知识库上下文 + 历史 + 用户消息)
2. LLM 调用含 tools 参数 → 模型决策是否调用工具
3. 若 tool_calls 非空 → 逐工具执行 → 结果追加到消息列表
4. 回填 tool 消息 + assistant(tool_calls) 消息 → 再次调 LLM
5. 重复 2-4，最多 5 轮
6. 最终 LLM 整合回复 → SSE 流式输出给前端
```

**记忆管理：**
- 短期记忆：保留最近 N 轮对话 (默认10轮)，超限后摘要压缩
- 长期记忆：持久化用户偏好/关键信息，每次对话注入
